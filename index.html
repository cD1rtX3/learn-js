Basic JavaScript Concepts

Introduction:

While learning to code is often said to be difficult, it's actually quite the opposite. Languages like JavaScript are built to be simple to understand and use, making learning easy with the right teacher.

Take this example code block:

let arr = [];
for (let i = 0; i < 10; i++) {
    arr[i] = i;
}

for (let i in arr) {
    let r = Number(i) + Math.trunc(Math.random() * (arr.length - Number(i)));
    let t = arr[i];
    arr[i] = arr[r];
    arr[r] = t;
}

It may look intimidating now, but breaking it down into its simple constructs yields an easy to follow pattern, and by the end of this module, you should be able to tell what this code is doing.

Your First Program:

Let's start with something a little less intimidating: a one-line program that prints the phrase “Hello, world!”.

// Hi!
console.log("Hello, world!");

Put this into a JavaScript interpreter (you can get one on this very site by pressing Ctrl+Shift+I and going into the console), and it will log that phrase to the console. Congratulations! You have written your very first program. Let's run down how it works.

Working from the outside in, the semicolon at the end tells JavaScript that you are done with your statement, like the period in English. It doesn't technically need to be there — the line ending tells enough — but examples here will have it included.

That green thing at the top is a comment. Comments are ignored by JavaScript, but are helpful for humans reading your code. Single-line comments start with // and go to the end of the line, and multi-line comments start with /* and end with */.

console.log is a function that prints out any arguments you give it. The curved brackets at the end enclose a list of arguments (though, in this case, there is only one).

The single argument given to console.log is the string "Hello, world!". The quotes around it tell JavaScript that it is a string.

Because of everything listed above, if you ran the code, you should see in the console:

Hello, world!
But What's That Underneath?

If you ran the code in the DevTools console (the one that pops up with Ctrl+Shift+I), you probably noticed that underneath our glorious message is:

undefined

What's that? As stated above, console.log is a function, and in JavaScript, functions have what is called a return value, which is just the value handed back by the function. In the case of console.log, the return value is simply undefined, which isn't very useful, but other functions exist that have cooler return values, like the function Math.sqrt:

Math.sqrt(9) // Returns 3

We'll get into functions more later, but for now, that's why the console says undefined.

Variables:

After saying hi to the world, the first step on our journey brings us to variables. Variables are how you store data in code. They are declared using the keyword let:

let foo;

You can use these just as you would any data:

// Prints "Hi!".
let foo = "Hi!";
console.log(foo);
