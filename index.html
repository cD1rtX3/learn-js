Basic JavaScript Concepts

Introduction:

It is a commonly held belief that learning to code is a difficult task. However, quite the opposite is the case. Languages like JavaScript are built to be simple to understand and use, making learning easy with the right teacher.

Take this example code block:

let arr = [];
for (let i = 0; i < 10; i++) {
    arr[i] = i;
}

for (let i in arr) {
    let r = i + Math.random() * (arr.length - i);
    let t = arr[i];
    arr[i] = arr[r];
    arr[r] = t;
}

It may look intimidating now, but breaking it down into its simple constructs yields an easy to follow pattern, and by the end of this module, you should be able to tell what this code is doing.

Basic Syntax:

Syntax is the bread and butter of a programming or scripting language. Many facets of JavaScript's syntax are situational and will be covered in their corresponding sections, but a few aren't and should be covered up here.

JavaScript is a “whitespace-agnostic” language, meaning it doesn't really care what whitespace is used, and often doesn't care that whitespace is used at all. This means that characters like tabulations and carriage returns are mostly just for humans reading the code.

Statements in JavaScript (and in most computer languages) end in a semicolon:

let foo = 1;

This tells JavaScript that you are finished with your statement, similar to the period in English.

Single-line comments are declared with two slashes, like this:

// This is a comment.

Multi-line comments are between a pair of /* and */, like this:

/*
    This is a multi-line comment. It can (as its name
    suggests) span multiple lines. It is useful for
    writing large blocks of text to help a reader
    understand your code.
*/

Variables:

Variables are how you store data within a script and are declared like this:

let foo = 0;

The let keyword, as you may have guessed, declares a variable. Variable names are case-sensitive (capitalization matters) and can be any string of letters, numbers, and/or underscores (_), though they cannot start with a number.

Perhaps an unintuitive facet of the language is the assignment (=) operator, which is a single equals sign. This is not to be confused (though it often is) with the equivalance operator (==, two equals signs). The assignment operator assigns a value to a variable and can be used in a let statement as above or on its own if the variable has already been declared:

foo = 2;

They can also be chained:

foo = bar = 3;

Multiple variables can be declared in a single statement, like this:

let a = 0, b = 1, c = 2;

and a variable does not have to be given a value:

let foo;

Data Types:

It's no fun to have variables if theres nothing to put in them. JavaScript has a few different types of things that can be put in a variable, one of which you have already seen — numbers;

Numbers:

// This is a number:
3

That was a number. Numbers are exactly what they sound like. They can look like this:

3

like this:

3.2

or even like this:

1_083.2

Strings:

A string is an amount of characters next to each other, like this:

"Hello!"

Note the quotes around it — those distinguish it from a variable name. They don't have to be double-quotes, either. They can be single-quotes:

'Hello!'

or graves (though grave-quotes have some special properties that we'll get to later):

`Hello!`

Boolean:

A boolean is a true or a false value, and looks like this:

true

or this:

false

Null and Undefined:

null is a purposefully-set “nothing” value:

null

while undefined is the value of an uninitialized variable (that is, one that hasn't had a value given to it):

undefined

The typeof Operator:

The typeof operator returns the type, as a string, of its operand. If you give it a number, it returns "number", if you give it a string, it returns "string", and so on. (Interestingly, typeof null returns "object".)

typeof 4 // "number"
typeof false // "boolean"
typeof undefined // "undefined"
